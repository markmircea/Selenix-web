Export Commands

These commands allow you to export scraped data to various file formats for further analysis or processing.

exportToCSV

Basic

Exports data to a CSV file. Works with arrays of objects, arrays of arrays, or simple arrays.

Syntax

exportToCSV | variableName | filePath

Parameters

Target: The name of the variable containing the data to export
Value: The file path where the CSV should be saved (relative or absolute)

Examples

scrapeStructured | {"_root": "css=.product", "name": "css=.name", "price": "css=.price", "stock": "css=.stock"} | products exportToCSV | products | products.csv
Export structured product data to CSV
scrapeCollection | css=.product-name | names scrapeCollection | css=.product-price | prices combineVariables | names,prices | combined exportToCSV | combined | products.csv
Combine and export multiple arrays to CSV
store | C:/exports/data_${new Date().toISOString().slice(0,10)}.csv | exportPath exportToCSV | results | ${exportPath}
Export to a file with a dynamic name including the current date

Data Format Handling

  • Arrays of Objects: Object keys become column headers
  • Arrays of Arrays: First array is used as headers if hasHeaders is true
  • Simple Arrays: Exported as a single column with no header

Advanced Options

For more control over the export format, you can pass a configuration object as the variable:

storeJson | { "data": products, "options": { "delimiter": ";", "hasHeaders": true, "columns": ["name", "price", "stock"], "quoteStrings": true } } | exportConfig exportToCSV | exportConfig | products.csv

Available Options

  • delimiter: Character to use as field separator (default: ",")
  • hasHeaders: Whether to include column headers (default: true)
  • columns: Specific columns to include and their order
  • quoteStrings: Whether to quote string values (default: true)
  • quoteChar: Character to use for quoting (default: '"')
  • escapeChar: Character to use for escaping (default: '"')
  • newline: Character to use for line endings (default: "\n")

exportToJSON

Basic

Exports data to a JSON file. Preserves the full structure of your data, including nested objects and arrays.

Syntax

exportToJSON | variableName | filePath

Parameters

Target: The name of the variable containing the data to export
Value: The file path where the JSON should be saved (relative or absolute)

Examples

scrapeStructured | {"_root": "css=.product", "name": "css=.name", "price": "css=.price", "details": {"brand": "css=.brand", "category": "css=.category"}} | products exportToJSON | products | products.json
Export complex nested data to JSON
storeJson | { "data": products, "metadata": { "source": "example.com", "timestamp": new Date().toISOString(), "count": products.length } } | exportData exportToJSON | exportData | products-with-metadata.json
Export data with additional metadata

Advanced Options

For more control over the export format, you can pass a configuration object as the variable:

storeJson | { "data": products, "options": { "pretty": true, "indent": 2 } } | exportConfig exportToJSON | exportConfig | products.json

Available Options

  • pretty: Whether to format the JSON with indentation (default: true)
  • indent: Number of spaces to use for indentation (default: 2)
  • replacer: Custom replacer function for JSON.stringify

exportToExcel

Advanced

Exports data to an Excel (.xlsx) file. Supports multiple worksheets, styling, and formatting options.

Syntax

exportToExcel | variableName | filePath

Parameters

Target: The name of the variable containing the data to export, or a configuration object
Value: The file path where the Excel file should be saved (relative or absolute)

Examples

scrapeStructured | {"_root": "css=.product", "name": "css=.name", "price": "css=.price", "stock": "css=.stock"} | products exportToExcel | products | products.xlsx
Export product data to Excel
storeJson | { "sheets": { "Products": products, "Categories": categories, "Summary": summary } } | workbook exportToExcel | workbook | report.xlsx
Export multiple sheets to a single Excel file
storeJson | { "data": products, "options": { "sheetName": "Product Data", "columns": [ { "header": "Product Name", "key": "name", "width": 30 }, { "header": "Price ($)", "key": "price", "width": 15 }, { "header": "In Stock", "key": "stock", "width": 15 } ], "headerStyle": { "font": { "bold": true, "color": { "argb": "FFFFFFFF" } }, "fill": { "type": "pattern", "pattern": "solid", "fgColor": { "argb": "FF4F46E5" } } } } } | excelConfig exportToExcel | excelConfig | formatted-products.xlsx
Export with custom formatting and styling

Advanced Features

  • Multiple Sheets: Export different datasets to separate worksheets
  • Column Customization: Define headers, widths, and data mappings
  • Styling: Apply fonts, colors, borders, and alignment
  • Formulas: Include Excel formulas in cells
  • Cell Formatting: Format numbers, dates, and percentages

Available Options

  • sheetName: Name of the worksheet (default: "Sheet1")
  • columns: Column definitions (header, key, width, style)
  • headerStyle: Style for the header row
  • rowStyle: Default style for data rows
  • alternateRowStyle: Style for alternate rows (zebra striping)
  • conditionalFormatting: Rules for applying styles based on cell values

exportToHTML

Advanced

Exports data to an HTML file. Perfect for creating reports with rich formatting and interactive elements.

Syntax

exportToHTML | variableName | filePath

Parameters

Target: The name of the variable containing the data to export, or a configuration object
Value: The file path where the HTML file should be saved (relative or absolute)

Examples

scrapeStructured | {"_root": "css=.product", "name": "css=.name", "price": "css=.price", "image": "css=img@src"} | products exportToHTML | products | products.html
Export product data to a simple HTML table
storeJson | { "data": products, "options": { "title": "Product Catalog", "template": "table", "styles": true, "sortable": true, "searchable": true } } | htmlConfig exportToHTML | htmlConfig | interactive-products.html
Export to an interactive HTML table with sorting and search
storeJson | { "data": products, "options": { "title": "Product Gallery", "template": "cards", "imageField": "image", "titleField": "name", "descriptionField": "description", "priceField": "price" } } | galleryConfig exportToHTML | galleryConfig | product-gallery.html
Export to a visual product gallery with images

Templates

  • table: Standard HTML table with optional sorting and filtering
  • cards: Visual card layout for items with images
  • list: Simple list format
  • custom: Use a custom HTML template

Interactive Features

  • sortable: Add column sorting functionality
  • searchable: Add search/filter functionality
  • pagination: Split large datasets into pages
  • responsive: Make the output mobile-friendly

Styling Options

  • styles: Include default CSS styles
  • theme: Choose from predefined themes ("light", "dark", "colorful")
  • customCSS: Add your own CSS styles

exportToText

Basic

Exports data to a plain text file. Useful for simple data or creating custom formatted outputs.

Syntax

exportToText | variableName | filePath

Parameters

Target: The name of the variable containing the data to export, or a configuration object
Value: The file path where the text file should be saved (relative or absolute)

Examples

scrapeCollection | css=.product-name | productNames exportToText | productNames | product-names.txt
Export a list of product names to a text file
storeJson | { "data": products, "options": { "format": "list", "fields": ["name", "price"], "separator": " - ", "lineEnding": "\r\n" } } | textConfig exportToText | textConfig | formatted-products.txt
Export with custom formatting
storeJson | { "data": products, "options": { "format": "custom", "template": "Product: ${name}\nPrice: ${price}\nIn Stock: ${stock}\n\n" } } | customConfig exportToText | customConfig | detailed-products.txt
Export using a custom template

Format Options

  • list: One item per line
  • table: ASCII table format
  • custom: Use a custom template string

Other Options

  • fields: Which fields to include in the output
  • separator: Character(s) to use between fields
  • lineEnding: Character(s) to use for line endings
  • template: Template string for custom format

exportScreenshot

Utility

Captures a screenshot of the current page and saves it to a file.

Syntax

exportScreenshot | filePath | options

Parameters

Target: The file path where the screenshot should be saved (relative or absolute)
Value: Optional JSON configuration for screenshot options

Examples

exportScreenshot | screenshot.png
Capture a screenshot of the entire viewport
exportScreenshot | product-section.png | {"selector": "#product-list"}
Capture a screenshot of a specific element
exportScreenshot | full-page.png | {"fullPage": true}
Capture a screenshot of the entire page, including content outside the viewport
store | screenshots/page_${new Date().toISOString().replace(/:/g, '-')}.png | screenshotPath exportScreenshot | ${screenshotPath}
Save screenshot with a timestamp in the filename

Screenshot Options

  • selector: CSS selector of the element to capture
  • fullPage: Whether to capture the full scrollable page (default: false)
  • quality: JPEG quality (0-100) for JPEG screenshots
  • omitBackground: Make background transparent for PNG screenshots
  • encoding: Image encoding ("binary", "base64")
  • clip: Clip rectangle: {x, y, width, height}

Supported Formats

  • PNG (.png)
  • JPEG (.jpg, .jpeg)
  • WebP (.webp)
