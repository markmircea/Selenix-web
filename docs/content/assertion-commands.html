Assertion & Verification Commands

These commands help you verify conditions, validate data, and ensure your automation is working correctly.

assertElementPresent

Basic

Verifies that an element is present on the page. Fails if the element is not found.

Syntax

assertElementPresent | locator

Parameters

Target: A locator that identifies the element to check

Examples

assertElementPresent | id=login-form
Verify that the login form is present
click | id=submit-button assertElementPresent | css=.success-message
Verify that a success message appears after clicking submit

Usage Notes

  • The command passes if at least one element matches the locator
  • The command fails if no elements match the locator
  • Use waitForElementPresent instead if you need to wait for an element to appear

Basic

Parameters

assertText

Basic

Verifies that an element's text matches the expected value. Fails if the text doesn't match.

Syntax

assertText | locator | expectedText

Parameters

Target: A locator that identifies the element to check
Value: The expected text

Examples

assertText | id=welcome-message | Welcome, User!
Verify welcome message text
click | id=add-to-cart assertText | id=cart-count | 1
Verify cart count after adding an item
store | Expected Title | expectedTitle assertText | css=h1 | ${expectedTitle}
Verify page title using a variable

Usage Notes

  • The comparison is exact and case-sensitive
  • Whitespace is included in the comparison
  • Use assertTextContains for partial text matching

Examples

Verify product description mentions "Waterproof"

assertValue

Basic

Verifies that an input element's value matches the expected value. Fails if the value doesn't match.

Syntax

assertValue | locator | expectedValue

Parameters

Target: A locator that identifies the input element to check
Value: The expected value

Examples

type | id=username | johndoe assertValue | id=username | johndoe
Verify input value after typing
click | id=reset-form assertValue | id=quantity | 1
Verify default value after form reset

Input Types

  • For text inputs, checks the current text value
  • For checkboxes/radio buttons, checks "on" or "off" depending on checked state
  • For select elements, checks the value of the selected option

assertSelected

Basic

Verifies that a specific option in a select element is selected. Fails if the option is not selected.

Syntax

assertSelected | selectLocator | optionLocator

Parameters

Target: A locator that identifies the select element
Value: The option to check (can be label, value, index, or id)

Examples

select | id=country | label=United States assertSelected | id=country | label=United States
Verify option is selected by label
select | id=product-size | value=M assertSelected | id=product-size | value=M
Verify option is selected by value
select | id=priority | index=0 assertSelected | id=priority | index=0
Verify first option is selected

Option Locator Types

  • label=Option Text: Match by visible text
  • value=optionValue: Match by option value attribute
  • index=n: Match by option index (0-based)
  • id=optionId: Match by option id attribute

assertAlert

Advanced

Verifies that an alert with the expected text is present. Fails if no alert is present or if the text doesn't match.

Syntax

assertAlert | expectedAlertText

Parameters

Target: The expected text of the alert

Examples

click | id=delete-button assertAlert | Are you sure you want to delete this item?
Verify delete confirmation alert
click | id=submit-form assertAlert | Form submitted successfully!
Verify success alert after form submission

Usage Notes

  • This command checks for JavaScript alert() dialogs, not custom UI alerts
  • The command automatically accepts (clicks OK) on the alert after verification
  • Use assertConfirmation for confirm() dialogs
  • Use assertPrompt for prompt() dialogs

assertConfirmation

Advanced

Verifies that a confirmation dialog with the expected text is present. Fails if no confirmation is present or if the text doesn't match.

Syntax

assertConfirmation | expectedConfirmationText

Parameters

Target: The expected text of the confirmation dialog

Examples

click | id=delete-account assertConfirmation | This action cannot be undone. Are you sure?
Verify account deletion confirmation

Usage Notes

  • This command checks for JavaScript confirm() dialogs, not custom UI confirmations
  • By default, the command accepts (clicks OK) on the confirmation after verification
  • Use chooseOkOnNextConfirmation or chooseCancelOnNextConfirmation before triggering the confirmation to control the response

assertPrompt

Advanced

Verifies that a prompt dialog with the expected text is present. Fails if no prompt is present or if the text doesn't match.

Syntax

assertPrompt | expectedPromptText

Parameters

Target: The expected text of the prompt dialog

Examples

click | id=rename-file assertPrompt | Enter new filename:
Verify rename prompt

Usage Notes

  • This command checks for JavaScript prompt() dialogs, not custom UI prompts
  • Use answerOnNextPrompt before triggering the prompt to provide an answer

assertAttribute

Basic

Verifies that an element's attribute has the expected value. Fails if the attribute doesn't exist or if the value doesn't match.

Syntax

assertAttribute | locator@attribute | expectedValue

Parameters

Target: A locator followed by @ and the attribute name (e.g., id=profile-image@src)
Value: The expected attribute value

Examples

assertAttribute | css=img.logo@alt | Company Logo
Verify image alt text
click | id=disable-button assertAttribute | id=submit-button@disabled | true
Verify button is disabled
assertAttribute | css=.error-field@class | form-control error
Verify element has specific classes

assertEval

Advanced

Verifies that the result of evaluating a JavaScript expression matches the expected value. Fails if the result doesn't match.

Syntax

assertEval | script | expectedValue

Parameters

Target: The JavaScript expression to evaluate
Value: The expected result

Examples

assertEval | document.title | My Dashboard
Verify page title using JavaScript
assertEval | document.querySelectorAll('.product-item').length | 10
Verify number of products on page
assertEval | window.isLoggedIn | true
Verify login state in JavaScript

Usage Notes

  • The script is executed in the context of the current page
  • The result is converted to a string for comparison
  • Complex objects are serialized to JSON strings
  • Use executeScript for scripts that don't need verification

verifyElementPresent

Basic

Checks if an element is present on the page. Unlike assert commands, verify commands continue execution even if the check fails.

Syntax

verifyElementPresent | locator

Parameters

Target: A locator that identifies the element to check

Examples

verifyElementPresent | id=optional-feature verifyElementPresent | css=.premium-content echo | Verification complete
Check for optional elements without stopping execution

Verify vs. Assert

  • Use verify commands when you want to check conditions without stopping execution
  • Use assert commands when the condition must be true to continue
  • Verify commands log failures but don't stop the test
  • Most assert commands have corresponding verify commands

waitForElementVisible

Basic

Waits for an element to be visible on the page. The element must be present in the DOM and visible (not hidden).

Syntax

waitForElementVisible | locator | [timeout]

Parameters

Target: A locator that identifies the element to wait for
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=show-details waitForElementVisible | id=details-panel
Wait for a panel to become visible after clicking a button
waitForElementVisible | css=.loading-indicator | 5000 waitForElementNotVisible | css=.loading-indicator
Wait for a loading indicator to appear and then disappear

Visibility Criteria

  • The element must be present in the DOM
  • The element must have non-zero dimensions
  • The element must not have display: none or visibility: hidden
  • The element must not be hidden by other elements

waitForElementNotVisible

Basic

Examples

Wait for a modal dialog to disappear after clicking close

Wait for a loading spinner to disappear after form submission

waitForText

Basic

Waits for an element's text to match the expected value.

Syntax

Parameters

Target: A locator that identifies the element to check
Value: The expected text
Timeout: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=refresh-count waitForText | id=counter | 10
Wait for a counter to reach a specific value
click | id=check-status waitForText | id=status | Complete | 30000
Wait up to 30 seconds for a status to change to "Complete"

waitForTextPresent

Basic

Waits for specific text to be present anywhere on the page.

Syntax

waitForTextPresent | text | [timeout]

Parameters

Target: The text to wait for
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=search-button waitForTextPresent | No results found
Wait for search results message to appear
click | id=load-data waitForTextPresent | Data loaded successfully | 15000
Wait up to 15 seconds for a success message

Usage Notes

  • This command searches the entire page text, not just a specific element
  • The search is case-sensitive
  • Use waitForText to check text in a specific element

Advanced

Waits for an element's attribute to have the expected value.

waitForCondition

Advanced

Waits for a JavaScript condition to evaluate to true.

Syntax

waitForCondition | script | [timeout]

Parameters

Target: The JavaScript condition to evaluate
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=load-data waitForCondition | return document.querySelectorAll('.data-row').length > 5
Wait for at least 5 data rows to be present
click | id=start-process waitForCondition | return window.dataLoaded === true | 60000
Wait up to 60 seconds for a JavaScript variable to be set
waitForCondition | return document.readyState === 'complete' && !document.querySelector('.loading')
Wait for page to load and loading indicator to disappear

Usage Notes

  • The script must return a boolean value (true or false)
  • The script is executed repeatedly until it returns true or the timeout is reached
  • The script must include a return statement
  • This is one of the most powerful wait commands for complex conditions

waitForElementEditable

Basic

Waits for an input element to become editable (enabled and not read-only).

Syntax

waitForElementEditable | locator | [timeout]

Parameters

Target: A locator that identifies the input element
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=edit-profile waitForElementEditable | id=username type | id=username | newusername
Wait for a field to become editable before typing
click | id=unlock-form waitForElementEditable | id=comments | 10000
Wait up to 10 seconds for a textarea to become editable

Editable Criteria

  • The element must be present and visible
  • The element must not have the disabled attribute
  • The element must not have the readonly attribute
