Templates & Sharing

Selenix templates allow you to share, reuse, and distribute complete automation workflows. Create reusable templates from your successful automations, or download pre-built templates for common tasks to accelerate your automation development.

Template System

Selenix templates include complete tests, suites, and configurations that can be exported as JSON files and shared across teams and projects.

Using Templates

Templates in Selenix are complete automation packages that include tests, test suites, and configurations. You can quickly import templates to get started with proven automation patterns.

Importing Templates

  1. Download template file - Get the .json template file from our library or community
  2. Open Selenix - Launch the application and open your project
  3. Import template - Go to File → Import → Import from JSON
  4. Select template file - Choose the downloaded template JSON file
  5. Review and customize - Adapt the template to your specific needs

Template Import Process

1. Select Import Type

Choose whether to import individual tests, complete suites, or entire project templates

2. Configure Settings

Adjust URLs, selectors, and variables to match your target website

3. Test & Validate

Run the imported template to ensure it works with your specific environment

Creating Templates

Turn your successful automations into reusable templates that can be shared with your team or the community.

Export Individual Tests

Create templates from single automation workflows:

Export Test to JSON Template

  1. Right-click on any test in your project
  2. Select "Export Test to JSON"
  3. Choose save location and filename
  4. Add description and metadata (optional)
Example Test Template Structure:
{
  "id": "web-scraper-template",
  "name": "E-commerce Product Scraper",
  "description": "Scrapes product data from e-commerce sites",
  "commands": [
    {
      "command": "open",
      "target": "${targetUrl}",
      "value": ""
    },
    {
      "command": "scrapeStructured", 
      "target": "{\"title\":\"css=.product-title\",\"price\":\"css=.price\"}",
      "value": "productData"
    },
    {
      "command": "exportToCSV",
      "target": "productData", 
      "value": "./exports/products.csv"
    }
  ],
  "variables": {
    "targetUrl": "https://example-shop.com/products"
  }
}

Export Test Suites

Create comprehensive templates with multiple related tests:

Export Current Suite

Export the currently active test suite with all its tests

  • Right-click on suite name
  • Select "Export Suite to JSON"
  • Includes all tests in the suite
  • Preserves test execution order

Export Multiple Suites

Create large templates with multiple test suites

  • Go to File → Export → Export Suites to JSON
  • Select specific suites to include
  • Creates comprehensive automation package
  • Perfect for complete workflow templates

Export All Suites

Export your entire project as a master template

  • File → Export → Export All Suites to JSON
  • Includes complete project structure
  • All tests, suites, and configurations
  • Ideal for project-wide templates

Template Best Practices

Use Variables

Replace hardcoded values with variables to make templates flexible:

  • URLs: ${baseUrl}
  • File paths: ${exportPath}
  • API keys: ${apiKey}
  • Selectors: ${productSelector}

Document Thoroughly

Include comprehensive documentation in your templates:

  • Clear test and suite names
  • Detailed descriptions
  • Setup instructions
  • Required variables list

Test Before Sharing

Ensure templates work in clean environments:

  • Test in new project
  • Verify all variables work
  • Check error handling
  • Validate output formats

Customizing Templates

After importing a template, customize it for your specific needs:

Variable Configuration

Setting Up Template Variables

// Original template variable
open | ${baseUrl}/products

// Customize for your site
open | https://mystore.com/products

// Use store command to set variables
store | https://mystore.com | baseUrl
store | css=.my-product-title | titleSelector
store | css=.my-price | priceSelector

Selector Adaptation

Adapting Selectors for Different Sites

Original Template:
scrapeCollection | css=.product-title | titles
Adapted for Your Site:
scrapeCollection | css=.item-name | titles
AI Assistant Help

Use the AI assistant to help adapt selectors: "Help me find the product title selector on this page"
