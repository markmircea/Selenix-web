Understanding Command Syntax

Every Selenix command follows this structure:

command | target | value

Command

The action to perform (e.g., click, type, scrapeCollection)

Target

The element locator, data, or configuration for the command

Value

Additional data, variable names, or options (often optional)

Element Selectors

Selenix provides multiple ways to identify elements on web pages:

id=elementId

Select by element ID (most reliable)

Example: id=submitButton

css=.class

Select using CSS selectors

Example: css=.product-title

xpath=//element

Select using XPath expressions

Example: xpath=//button[contains(text(),'Submit')]

name=fieldName

Select by name attribute

Example: name=username

linkText=Link Text

Select links by exact text

Example: linkText=Click Here

partialLinkText=Partial

Select links by partial text

Example: partialLinkText=Click

Working with Variables

Selenix supports different variable types for storing and processing data:

String Variables

Store text data: ${userName}

Created by: store, storeText, storeValue

Array Variables

Store multiple items: ${productList}

Created by: scrapeCollection, importFromCSV

Object Variables

Store structured data: ${userProfile.email}

Created by: storeJson, scrapeStructured, httpRequest

Number Variables

Store numeric data: ${itemCount}

Created by: storeElementCount, calculations
