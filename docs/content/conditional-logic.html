Conditional Logic

Conditional logic in Selenix allows your automations to make intelligent decisions based on page content, variable values, and dynamic conditions. Create smart workflows that adapt to different scenarios automatically.

AI-Assisted Logic

Ask the AI assistant: "Help me create a condition to check if a product is in stock" and it will suggest the optimal logic structure.

Basic Conditional Statements

Selenix provides powerful conditional logic commands for creating intelligent automations:

if Statement

Execute commands only when a condition is true

if | ${price} > 100 |
Executes following commands if price variable is greater than 100

else Statement

Execute alternative commands when if condition is false

else | |
Provides alternative path when if condition fails

elseIf Statement

Test additional conditions in sequence

elseIf | ${stock} === "available" |
Tests another condition if previous conditions were false

end Statement

Close conditional blocks

end | |
Terminates if/else/elseIf blocks

Condition Types

Selenix supports various types of conditions for different scenarios:

Value Comparisons

${price} > 50 Greater than comparison
${quantity} <= 10 Less than or equal comparison
${status} === "active" Exact equality check
${category} !== "discontinued" Not equal comparison

Text Conditions

${title}.includes("Special") Text contains substring
${email}.endsWith("@company.com") Text ends with pattern
${name}.startsWith("Mr.") Text starts with pattern
${description}.length > 100 Text length comparison

Element Existence

css=.add-to-cart Check if element exists on page
xpath=//button[text()="Buy Now"] Check for button with specific text
id=login-form Check if login form is present

Array & Object Conditions

${products}.length > 0 Check if array has items
${user.role} === "admin" Check object property value
${categories}.includes("electronics") Check if array contains value
