Parameters

Target: Number of times to scroll down the page
Value: Wait time in milliseconds between each scroll

Examples

scrollAndWait | 5 | 2000 scrapeCollection | css=.infinite-scroll-item | allItems scrollAndWait | 10 | 1000 storeElementCount | css=.loaded-item | totalLoaded
Scroll to load more content, then scrape all items

Best Practices

  • Use with social media feeds, search results, and product listings

  • Adjust wait times based on site loading speed
  • Use before scraping to ensure all content is visible

monitorElement

Professional

Periodically checks an element for changes to a specified attribute and performs actions when changes are detected. Perfect for monitoring dynamic content.

Syntax

monitorElement | locator | jsonConfig

Parameters

Target: Element locator to monitor
Value: JSON configuration with monitoring options

Examples

monitorElement | css=.stock-price | {"interval": 1000, "storeValue": "newPrice"} monitorElement | id=status | {"attribute": "class", "condition": "contains", "baseValue": "complete"} monitorElement | css=.countdown | {"interval": 500, "condition": "===", "baseValue": "00:00"}
Monitor stock prices, loading status, and countdown timers

Configuration Options

  • interval: Milliseconds between checks (default: 5000)
  • attribute: What to monitor - "text", "value", "class", etc. (default: "text")
  • condition: "!==", "===", ">", "<", "contains", "matches"
  • storeValue: Variable to store the changed value
  • maxDuration: Maximum monitoring time in milliseconds
