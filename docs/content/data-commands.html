Data Management Commands

These commands help you store, manipulate, and process data during your automation workflows.

store

Basic

Save a target string as a variable for easy re-use. Variable will be string type.

Syntax

store | text | variableName

Parameters

Target: The text to store
Value: The name of the variable to store the text in

Examples

store | John Doe | username store | password123 | password type | id=username | ${username} type | id=password | ${password}
Store login credentials for reuse
store | https://example.com/api | baseUrl store | ${baseUrl}/users | usersEndpoint store | ${baseUrl}/products | productsEndpoint
Build URLs from a base URL

storeJson

Advanced

Save JSON as an object on a variable. Variable will be object type, allowing you to access properties directly.

Syntax

storeJson | json | variableName

Parameters

Target: JSON string or object literal
Value: The name of the variable to store the JSON object in

Examples

storeJson | {"name": "John", "age": 30, "roles": ["admin", "user"]} | user echo | User: ${user.name}, Age: ${user.age} echo | First role: ${user.roles[0]}
Store and access a JSON object
storeJson | [1, 2, 3, 4, 5] | numbers echo | First number: ${numbers[0]} echo | Last number: ${numbers[4]}
Store and access a JSON array
storeJson | {"config": {"timeout": 5000, "retries": 3}, "endpoints": {"users": "/api/users", "products": "/api/products"}} | settings echo | Timeout: ${settings.config.timeout}ms echo | Users endpoint: ${settings.endpoints.users}
Store and access nested JSON objects

Usage Notes

  • JSON must be valid - use proper quotes, commas, and brackets
  • Access object properties using dot notation: ${variable.property}
  • Access array elements using bracket notation: ${variable[index]}
  • Combine with transformVariable for advanced data manipulation

storeValue

Basic

Gets the value of an input element and stores it for later use. Variable will be string type.

Syntax

storeValue | locator | variableName

Parameters

Target: A locator that identifies the input element
Value: The name of the variable to store the value in

Examples

storeValue | id=quantity | quantity echo | Current quantity: ${quantity}
Get the value of a number input
storeValue | id=username | enteredUsername storeValue | id=password | enteredPassword echo | Username: ${enteredUsername}, Password: ${enteredPassword}
Get values from a login form

Input Types

  • For text inputs, gets the current text value
  • For checkboxes/radio buttons, gets "on" or "off" depending on checked state
  • For select elements, gets the value of the selected option

storeTitle

Basic

Gets the title of the current page. Variable will be string type.

Syntax

storeTitle | variableName

Parameters

Target: The name of the variable to store the page title in

Examples

storeTitle | pageTitle echo | Current page: ${pageTitle}
Get and display the current page title
storeTitle | beforeTitle click | id=next-page waitForElementPresent | id=content storeTitle | afterTitle echo | Navigation: ${beforeTitle} -> ${afterTitle}
Track page navigation through title changes

storeWindowHandle

Advanced

Gets the handle of the current page. Useful for managing multiple windows or tabs.

Syntax

storeWindowHandle | variableName

Parameters

Target: The name of the variable to store the window handle in

Examples

storeWindowHandle | mainWindow click | id=open-new-window selectWindow | win_ser_1 storeWindowHandle | popupWindow // Do something in the popup selectWindow | ${mainWindow}
Switch between windows and return to the original window

combineVariables

Advanced

Combines multiple variables into one variable with intelligent type handling. Perfect for preparing data for export operations.

Syntax

combineVariables | variablesList | resultVariableName

Parameters

Target: A comma-separated list of variable names to combine, or a JSON configuration for advanced options
Value: The name of the variable to store the combined result

Examples

scrapeCollection | css=.product-name | names scrapeCollection | css=.product-price | prices scrapeCollection | css=.product-stock | stock combineVariables | names,prices,stock | products exportToCSV | products | products.csv
Combine separate arrays into a single dataset for export
combineVariables | {"variables":["users","orders"],"mode":"object","options":{"useName":true}} | userData
Advanced combination with explicit mode and options
store | John | firstName store | Doe | lastName combineVariables | {"variables":["firstName","lastName"],"mode":"string","options":{"separator":" "}} | fullName echo | Full name: ${fullName}
Combine strings with a separator

Combination Modes

  • Array Mode: Combines values into an array. Arrays are concatenated.
  • Object Mode: Combines values into an object. Objects are merged.
  • String Mode: Combines values into a string with optional separator.
  • Auto-detection: If mode is not specified, the command automatically determines the best mode based on variable types.

Advanced Options

  • Array Mode Options:
    • objectToEntries: When true, converts object variables to arrays of key-value pairs
  • Object Mode Options:
    • keyConflict: How to handle key conflicts ("override", "rename", "error")
    • keyPrefix: Prefix to use for keys when adding primitive values
    • useName: When true, uses the variable name as the key for primitive values
  • String Mode Options:
    • separator: String to use between combined values (default: "")
    • nullHandling: How to handle null/undefined values ("include" or "exclude")

inspectVariable

Utility

Display detailed information about a variable including its type and structure. Useful for debugging and data exploration.

Syntax

inspectVariable | variableName

Parameters

Target: The name of the variable to inspect

Examples

scrapeStructured | {"_root": "css=.product", "name": "css=.name", "price": "css=.price"} | products inspectVariable | products
Inspect scraped data structure
storeJson | {"user": {"name": "John", "roles": ["admin", "editor"]}} | userData inspectVariable | userData
Inspect a complex JSON object

Output Information

  • Variable type (string, number, array, object, etc.)
  • Array length (for arrays)
  • Object keys (for objects)
  • Sample values (for large arrays/objects)
  • Nested structure visualization

httpRequest

Advanced

Sends an HTTP request and stores the response in a variable. Perfect for API testing and data retrieval.

Syntax

httpRequest | requestConfig | variableName

Parameters

Target: JSON configuration for the HTTP request (URL, method, headers, body, etc.)
Value: Optional variable name to store the result

Examples

httpRequest | {"url": "https://api.example.com/users", "method": "GET"} | users echo | Retrieved ${users.length} users
Simple GET request
httpRequest | {"url": "https://api.example.com/login", "method": "POST", "headers": {"Content-Type": "application/json"}, "body": {"username": "test", "password": "password"}} | loginResponse store | ${loginResponse.token} | authToken
POST request with JSON body
store | my-api-key | apiKey httpRequest | {"url": "https://api.example.com/data", "method": "GET", "headers": {"Authorization": "Bearer ${apiKey}"}} | apiData
Request with authorization header using a variable

Request Configuration Options

  • url: The URL to send the request to (required)
  • method: HTTP method (GET, POST, PUT, DELETE, etc.)
  • headers: Object containing request headers
  • body: Request body (object for JSON, string for other formats)
  • params: URL parameters as an object
  • timeout: Request timeout in milliseconds
  • responseType: Expected response type (json, text, blob, etc.)
