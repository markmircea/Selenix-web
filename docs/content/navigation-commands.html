Navigation & Flow Control Commands

These commands help you navigate between pages, control execution flow, and manage browser windows and tabs.

open

Basic

Opens a URL in the current browser window. This is typically the first command in a test.

Syntax

open | url

Parameters

Target: The URL to open

Examples

open | https://example.com
Open a website with absolute URL
open | /products
Open a page with relative URL
store | https://example.com | baseUrl open | ${baseUrl}/products
Open a URL using a variable

Usage Notes

  • Absolute URLs must include the protocol (http:// or https://)
  • Relative URLs are resolved against the current base URL
  • The command waits for the page to load before continuing
  • Use setTimeout to adjust the page load timeout if needed

debugger

Advanced

Breaks the execution and enters debugger mode. Useful for debugging test scripts.

Syntax

debugger

Examples

open | https://example.com debugger click | id=submit-button
Pause execution to inspect the page state

Usage Notes

  • Execution will pause at this command when running in debug mode
  • Use this to inspect variables, page state, or step through commands
  • Remove debugger commands before running tests in production

times

Advanced

Create a loop that executes the proceeding commands n number of times, can end with END.

Syntax

times | numberOfTimes | [loopLimit] // commands to execute repeatedly end

Parameters

Target: The number of times to execute the loop
Value: Optional loop limit for safety (should match or exceed the target)

Examples

times | 5 click | css=.next-button pause | 1000 scrapeCollection | css=.data-item | pageData end
Click next button and scrape data exactly 5 times
times | 3 click | id=refresh-button waitForElementVisible | css=.updated-content echo | Refresh attempt completed end
Refresh content 3 times with status logging

Usage Notes

  • Executes exactly the specified number of times
  • More predictable than while loops for known iterations
  • Use when you know exactly how many times to repeat
  • Loop counter is available as ${index} (0-based)

forEach

Advanced

Create a loop that executes the proceeding commands for each item in a given collection. Must be terminated with an "end" command.

Syntax

forEach | arrayVariableName | iteratorVariableName // commands to execute for each item end

Parameters

Target: The name of a variable containing an array to iterate over
Value: The name to assign to each item during iteration

Examples

storeJson | ["page1", "page2", "page3"] | pages forEach | pages | page open | https://example.com/${page} scrapeCollection | css=.content | data echo | Scraped data from ${page} end
Iterate through an array of page names
scrapeCollection | css=.product-link@href | productUrls forEach | productUrls | url open | ${url} storeText | css=.price | price echo | Product price: ${price} end
Visit each product URL and extract pricing information

Usage Notes

  • The array variable must exist and contain an array
  • The iterator variable (${item}) is available inside the loop
  • Loop index is available as ${index} (0-based)
  • Perfect for processing scraped collections of data

do

Advanced

Create a loop that executes the proceeding commands at least once. Terminate the branch with the repeat if command.

Syntax

do // commands to execute repeatIf | condition

Examples

store | 1 | page do open | https://example.com/page/${page} scrapeCollection | css=.item | items storeEval | ${page} + 1 | page repeatIf | ${page} <= 5
Scrape pages 1 through 5 using a do-while loop

Usage Notes

  • The commands inside the do block execute at least once
  • Use repeatIf to specify the condition for continuing the loop
  • Ensure the condition will eventually become false to avoid infinite loops

while

Advanced

Create a loop that executes the proceeding commands repeatedly for as long as the provided conditional expression is true. End with END.

Syntax

while | conditionalExpression | [loopLimit] // commands to execute repeatedly end

Parameters

Target: A JavaScript expression that evaluates to true or false
Value: Optional maximum number of iterations to prevent infinite loops

Examples

store | 1 | page while | ${page} <= 5 open | https://example.com/page/${page} scrapeCollection | css=.item | items storeEval | ${page} + 1 | page end
Loop through multiple pages while condition is true
storeElementPresent | css=.load-more | hasMore while | ${hasMore} | 10 click | css=.load-more pause | 2000 storeElementPresent | css=.load-more | hasMore end
Keep clicking "load more" while it exists (max 10 times)

Loop Safety

  • Always include a loop limit to prevent infinite loops
  • Ensure the condition will eventually become false
  • Update variables inside the loop that affect the condition
  • Use echo commands to monitor loop progress

if

Advanced

Create a conditional branch in your test. Terminate the branch with the end command.

Syntax

if | conditionalExpression // commands to execute if true end

Parameters

Target: A JavaScript expression that evaluates to true or false

Examples

storeElementPresent | id=login-form | isLoginPage if | ${isLoginPage} echo | User needs to log in type | id=username | testuser click | id=login-button end
Conditionally perform login if login form is present
storeText | id=item-count | count if | ${count} > 10 echo | Many items found: ${count} click | id=show-all-button end
Execute commands based on numeric comparison

Conditional Expressions

  • Use JavaScript syntax: ==, !=, >, <, >=, <=
  • String comparisons: ${text} == "expected"
  • Boolean variables: ${isPresent} or !${isPresent}
  • Combine with && (and) or || (or) operators

else

Advanced

Part of an if block. Execute the commands in this branch when an if and/or else if condition are not met. Terminate the branch with the end command.

Syntax

if | condition // commands if true else // commands if false end

Examples

storeElementPresent | id=login-form | isLoginPage if | ${isLoginPage} echo | User needs to log in type | id=username | testuser click | id=login-button else echo | User is already logged in end
Handle different page states with if-else

elseIf

Advanced

Part of an if block. Execute the commands in this branch when an if condition has not been met. Terminate the branch with the end command.

Syntax

if | condition1 // commands if condition1 is true elseIf | condition2 // commands if condition2 is true else // commands if all conditions are false end

Parameters

Target: The condition to evaluate (JavaScript expression)

Examples

storeText | id=status | currentStatus if | ${currentStatus} == "pending" echo | Order is pending elseIf | ${currentStatus} == "shipped" echo | Order has been shipped elseIf | ${currentStatus} == "delivered" echo | Order has been delivered else echo | Unknown status: ${currentStatus} end
Handle multiple conditions with if-elseIf-else

end

Advanced

Terminates a control flow block for if, while, times, and forEach commands.

Syntax

end

Examples

if | ${condition} // commands end while | ${condition} // commands end times | 5 // commands end
End different types of control flow blocks

Usage Notes

  • Every control flow command (if, while, times, forEach) must have a matching end
  • Proper nesting is required - end commands close the most recent open block
  • Missing end commands will cause syntax errors

close

Basic

Closes the current window. There is no need to close the initial window, IDE will re-use it; closing it may cause a performance penalty on the test.

Syntax

close

Examples

click | id=open-popup selectWindow | win_ser_1 // Work with popup close selectWindow | win_ser_local
Close a popup window and return to main window

Usage Notes

  • Closes the currently active window or tab
  • Avoid closing the main window unless necessary
  • Use selectWindow to switch back to remaining windows

echo

Utility

Prints the specified message to the log. Essential for debugging and monitoring automation progress.

Syntax

echo | message

Examples

echo | Starting product scraping process echo | Found ${productCount} products on page ${currentPage} echo | Processing user: ${user.name} (${user.email}) echo | Export completed: ${exportedRecords} records saved
Log progress, variable values, and completion status

Debugging Tips

  • Use before/after scraping commands to track progress
  • Include variable values to verify data processing
  • Add timestamps: ${new Date().toISOString()}
  • Use in loops to monitor iteration progress

pause

Utility

Wait for the specified amount of time before continuing execution.

Syntax

pause | milliseconds

Examples

click | css=.load-more-button pause | 3000 scrapeCollection | css=.newly-loaded-item | newItems pause | 5000 click | id=submit-form
Wait for content to load and forms to process

When to Use Pause

  • After clicking load buttons to wait for new content
  • Between rapid API calls to respect rate limits
  • For slow-loading dynamic content
  • Consider using waitForElementVisible instead when possible

executeAsyncScript

Advanced

Executes an async snippet of JavaScript in the context of the currently selected frame or window. The script fragment will be executed as the body of an anonymous function and must return a Promise.

Syntax

executeAsyncScript | script | [variableName]

Parameters

Target: Async JavaScript code to execute. Must include the "callback" parameter and call it when done
Value: Optional variable name to store the Promise result

Examples

executeAsyncScript | const result = await fetch('/api/data'); callback(result.json()); | apiData echo | API response: ${apiData}
Fetch data from an API asynchronously
executeAsyncScript | return new Promise(resolve => setTimeout(() => resolve('done'), 1000)) | result echo | Async operation result: ${result}
Execute an async operation with Promise

Usage Notes

  • The script must handle asynchronous operations properly
  • Use callback parameter for traditional async patterns
  • Use return Promise for modern async/await patterns
  • The Promise result will be saved in the variable if specified

repeatIf

Advanced

Terminate a 'do' control flow branch conditionally. If the result of the provided conditional expression is true, it starts the do loop over. Otherwise it ends the loop.

Syntax

do // commands to execute repeatIf | condition | [loopLimit]

Parameters

Target: The condition to evaluate for continuing the loop
Value: Optional maximum number of iterations to prevent infinite loops

Examples

store | 1 | attempts do click | id=submit-button pause | 1000 storeElementPresent | css=.error-message | hasError storeEval | ${attempts} + 1 | attempts repeatIf | ${hasError} && ${attempts} < 3
Retry form submission up to 3 times if errors occur

run

Advanced

Runs a test case from the current project.

Syntax

run | testCaseName

Parameters

Target: The name of the test case to run

Examples

run | LoginTest run | SetupUserData // Continue with main test
Run prerequisite test cases before main test

Usage Notes

  • The test case must exist in the current project
  • Variables and state are shared between test cases
  • Use this for modular test design and reusable test components

runScript

Advanced

Creates a new "script" tag in the body of the current test window, and adds the specified text into the body of the command. Beware that JS exceptions thrown in these script tags aren't managed by Selenium, so you should probably wrap your script in try/catch blocks if there is any chance that the script will throw an exception.

Syntax

runScript | script

Parameters

Target: JavaScript code to inject into the page as a script tag

Examples

runScript | console.log('Script injected successfully');
Inject a simple logging script
runScript | try { window.myCustomFunction = function() { return 'Hello World'; }; } catch(e) { console.error(e); }
Inject a custom function with error handling

Usage Notes

  • Creates a permanent script tag in the page
  • Script persists for the lifetime of the page
  • Use try/catch blocks to handle potential exceptions
  • Different from executeScript which runs immediately and returns values

executeScript

Advanced

Executes a snippet of JavaScript in the context of the currently selected frame or window. The script fragment will be executed as the body of an anonymous function. To store the return value, use the 'return' keyword and provide a variable name in the value input field.

Syntax

executeScript | script | [variableName]

Parameters

Target: JavaScript code to execute. Can access page elements and variables
Value: Optional variable name to store the result (only needed if script returns a value)

Examples

executeScript | return document.title | pageTitle echo | Page title is: ${pageTitle}
Get the page title and store it in a variable
executeScript | window.scrollTo(0, 500)
Scroll the page to a specific position
executeScript | return document.querySelector('.price').textContent | price echo | Product price: ${price}
Extract text content from an element

Usage Notes

  • Use 'return' keyword to capture values in variables
  • Can access and modify page elements directly
  • Useful for complex operations not available in standard commands
  • Script runs in the context of the current page

acceptAlert

Advanced

Affects a currently showing alert. This command instructs Selenium to accept it.

Syntax

acceptAlert

Examples

click | id=delete-button acceptAlert
Accept a delete confirmation alert
click | id=submit-form assertAlert | Form submitted successfully! acceptAlert
Verify alert text and then accept it

Usage Notes

  • Works with JavaScript alert() dialogs
  • Equivalent to clicking the "OK" button
  • Use assertAlert first to verify the alert text
  • Alert must be currently showing when this command executes

dismissPrompt

Advanced

Affects a currently showing alert prompt. This command instructs Selenium to dismiss it.

Syntax

dismissPrompt

Examples

click | id=rename-file dismissPrompt
Cancel a file rename prompt dialog
click | id=add-comment assertPrompt | Enter your comment: dismissPrompt echo | User cancelled comment entry
Verify prompt text and then dismiss it

Usage Notes

  • Works with JavaScript prompt() dialogs
  • Equivalent to clicking the "Cancel" button
  • Use answerPrompt to provide an answer instead
  • Prompt dialog must be currently showing

acceptConfirmation

dismissConfirmation

Cancel a delete account confirmation dialog

setSpeed

Basic

Set execution speed (e.g., set the millisecond length of a delay which will follow each Selenium operation). By default, there is no such delay.

Syntax

setSpeed | milliseconds

Parameters

Target: The delay in milliseconds to add after each command

Examples

setSpeed | 1000 click | id=button1 click | id=button2 setSpeed | 0
Add 1 second delay between commands, then reset to no delay
setSpeed | 500 forEach | urls | url open | ${url} scrapeCollection | css=.data | data end setSpeed | 0
Add delay when scraping multiple pages to avoid rate limiting

Usage Notes

  • This setting is global and affects all subsequent commands
  • Use this to slow down execution for demonstration or to avoid rate limiting
  • Set to 0 to remove delays and run at full speed
  • The delay is added after each command execution

setWindowSize

Advanced

Set the browser's window size, including the browser's interface.

Syntax

setWindowSize | resolution

Parameters

Target: The window size in format "widthxheight" (e.g., "1920x1080")

Examples

setWindowSize | 1920x1080 open | https://example.com
Set window to full HD resolution before opening a page
setWindowSize | 375x667 // Test mobile layout
Set window to mobile device size for responsive testing

Common Resolutions

  • Desktop: 1920x1080, 1366x768, 1440x900
  • Mobile: 375x667 (iPhone), 360x640 (Android)
  • Tablet: 768x1024 (iPad), 1024x768 (iPad landscape)

answerPrompt

Advanced

Affects a currently showing alert prompt. This command instructs Selenium to provide the specified answer to it.

Syntax

answerPrompt | answer

Parameters

Target: The answer to provide to the prompt dialog

Examples

click | id=rename-file answerPrompt | new-filename.txt
Provide an answer to a JavaScript prompt dialog

Usage Notes

  • This command works with JavaScript prompt() dialogs
  • The prompt must be currently showing when this command is executed
  • Use assertPrompt to verify the prompt text before answering
  • The command automatically clicks OK after providing the answer

selectWindow

Advanced

Switches focus to a different window or tab. Useful for working with multiple windows or popup windows.

Syntax

selectWindow | windowLocator

Parameters

Target: The window to select, specified by handle, name, title, or index

Examples

storeWindowHandle | mainWindow click | id=open-popup selectWindow | win_ser_1 // Work with the popup window selectWindow | ${mainWindow}
Work with a popup window and return to the main window
selectWindow | title=Product Details
Select a window by its title
selectWindow | index=1
Select the second window (0-based index)

Window Locator Types

  • handle=windowHandle: Select by window handle (from storeWindowHandle)
  • name=windowName: Select by window name
  • title=windowTitle: Select by window title
  • index=n: Select by window index (0-based)
  • win_ser_n: Select by window serial number (1-based, assigned in order of opening)

waitForElementPresent

Basic

Waits for an element to be present in the DOM. Does not require the element to be visible.

Syntax

waitForElementPresent | locator | [timeout]

Parameters

Target: A locator that identifies the element to wait for
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=load-data waitForElementPresent | id=data-table
Wait for a table to be loaded after clicking a button
click | id=show-more waitForElementPresent | css=.additional-content | 15000
Wait up to 15 seconds for additional content to appear

Presence vs. Visibility

  • This command only checks if the element exists in the DOM
  • The element does not need to be visible to satisfy this wait
  • Use waitForElementVisible if you need to wait for an element to be visible

waitForElementNotPresent

Basic

Waits for an element to be removed from the DOM.

Syntax

waitForElementNotPresent | locator | [timeout]

Parameters

Target: A locator that identifies the element to wait for
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=delete-item waitForElementNotPresent | id=item-123
Wait for an item to be removed after clicking delete
click | id=submit-form waitForElementNotPresent | css=.loading-indicator | 20000
Wait up to 20 seconds for a loading indicator to disappear
