Navigation & Flow Control Commands

These commands help you navigate between pages, control execution flow, and manage browser windows and tabs.

open

Basic

Opens a URL in the current browser window. This is typically the first command in a test.

Syntax

open | url

Parameters

Target: The URL to open

Examples

open | https://example.com
Open a website with absolute URL
open | /products
Open a page with relative URL
store | https://example.com | baseUrl open | ${baseUrl}/products
Open a URL using a variable

Usage Notes

  • Absolute URLs must include the protocol (http:// or https://)
  • Relative URLs are resolved against the current base URL
  • The command waits for the page to load before continuing
  • Use setTimeout to adjust the page load timeout if needed

goBack

Basic

Simulates clicking the browser's back button. Navigates to the previous page in the browser history.

Syntax

goBack

Examples

open | https://example.com click | linkText=Products goBack assertTitle | Example Domain
Navigate to a page and then back to the previous page

goForward

Basic

Simulates clicking the browser's forward button. Navigates forward in the browser history.

Syntax

goForward

Examples

open | https://example.com click | linkText=Products goBack goForward assertTitle | Products - Example.com
Navigate back and then forward again

refresh

Basic

Refreshes the current page. Equivalent to clicking the browser's refresh button.

Syntax

refresh

Examples

open | https://example.com refresh
Refresh the current page
click | id=update-data refresh assertElementPresent | css=.updated-indicator
Update data and refresh to see changes

if

Advanced

Begins a conditional block of commands. The commands inside the block are executed only if the condition is true.

Syntax

if | condition // commands to execute if condition is true [else] // commands to execute if condition is false end

Parameters

Target: The condition to evaluate (JavaScript expression)

Examples

storeElementPresent | id=login-form | loginFormPresent if | ${loginFormPresent} type | id=username | johndoe type | id=password | password123 click | id=login-button else echo | Already logged in end
Log in only if the login form is present
storeText | id=item-count | count if | ${count} > 0 click | id=checkout else echo | Cart is empty end
Proceed to checkout only if items are in the cart
storeEval | new Date().getHours() | currentHour if | ${currentHour} < 12 echo | Good morning! else if | ${currentHour} < 18 echo | Good afternoon! else echo | Good evening! end end
Nested if statements for time-based greeting

Condition Syntax

  • Conditions can use variables: ${varName} == "value"
  • Comparison operators: ==, !=, >, <, >=, <=
  • Logical operators: && (and), || (or), ! (not)
  • JavaScript expressions: ${varName}.includes("text"), ${count} % 2 == 0

Structure

  • Each if must have a matching end
  • The else block is optional
  • Conditions can be nested

while

Advanced

Begins a loop that executes commands repeatedly as long as the condition is true.

Syntax

while | condition // commands to execute while condition is true end

Parameters

Target: The condition to evaluate (JavaScript expression)

Examples

store | 1 | page while | ${page} <= 5 open | https://example.com/products?page=${page} scrapeCollection | css=.product-title | titles echo | Scraped page ${page} storeEval | ${page} + 1 | page end
Scrape multiple pages of products
storeElementCount | css=.pagination-next:not(.disabled) | hasNextPage while | ${hasNextPage} > 0 scrapeCollection | css=.product-item | products click | css=.pagination-next waitForPageToLoad storeElementCount | css=.pagination-next:not(.disabled) | hasNextPage end
Scrape all pages until there's no next page

Important Considerations

  • Always ensure the condition will eventually become false to avoid infinite loops
  • Use a counter or timeout mechanism as a safety measure
  • The condition is evaluated before each iteration
  • Each while must have a matching end

forEach

Advanced

Iterates over each item in an array variable, executing the commands in the loop for each item.

Syntax

forEach | arrayVariable | itemVariable // commands to execute for each item end

Parameters

Target: The name of the array variable to iterate over
Value: The name to use for the current item variable

Examples

scrapeCollection | css=.product-url | productUrls forEach | productUrls | url open | ${url} storeText | css=.product-title | title storeText | css=.product-price | price echo | Product: ${title}, Price: ${price} end
Visit each product page and extract details
storeJson | ["apple", "banana", "cherry"] | fruits forEach | fruits | fruit type | id=search-box | ${fruit} click | id=search-button waitForElementPresent | css=.results storeElementCount | css=.result-item | resultCount echo | Found ${resultCount} results for ${fruit} end
Search for each item in a list

Usage Notes

  • The array variable must contain an array or array-like object
  • The item variable is created automatically for each iteration
  • You can access the current index using ${_index} inside the loop
  • Each forEach must have a matching end

times

Basic

Executes a block of commands a specified number of times.

Syntax

times | count // commands to repeat end

Parameters

Target: The number of times to repeat the commands

Examples

times | 5 click | id=add-to-cart pause | 500 end assertText | id=cart-count | 5
Add 5 items to the cart
store | 3 | numPages times | ${numPages} echo | Processing page ${_index + 1} of ${numPages} scrapeCollection | css=.item | items click | css=.next-page waitForPageToLoad end
Process a specific number of pages

Usage Notes

  • The current iteration index is available as ${_index} (0-based)
  • The count can be a number or a variable containing a number
  • Each times must have a matching end

selectWindow

Advanced

Switches focus to a different window or tab. Useful for working with multiple windows or popup windows.

Syntax

selectWindow | windowLocator

Parameters

Target: The window to select, specified by handle, name, title, or index

Examples

storeWindowHandle | mainWindow click | id=open-popup selectWindow | win_ser_1 // Work with the popup window selectWindow | ${mainWindow}
Work with a popup window and return to the main window
selectWindow | title=Product Details
Select a window by its title
selectWindow | index=1
Select the second window (0-based index)

Window Locator Types

  • handle=windowHandle: Select by window handle (from storeWindowHandle)
  • name=windowName: Select by window name
  • title=windowTitle: Select by window title
  • index=n: Select by window index (0-based)
  • win_ser_n: Select by window serial number (1-based, assigned in order of opening)

close

Basic

Closes the current window or tab. If this is the last open window, the browser will remain open with a blank page.

Syntax

close

Examples

storeWindowHandle | mainWindow click | id=open-popup selectWindow | win_ser_1 // Work with the popup window close selectWindow | ${mainWindow}
Work with a popup window and close it

setTimeout

Basic

Sets the timeout for subsequent commands. Affects wait commands and implicit waits.

Syntax

setTimeout | milliseconds

Parameters

Target: The timeout value in milliseconds

Examples

setTimeout | 10000 open | https://slow-loading-site.com
Increase timeout for a slow-loading site
setTimeout | 30000 waitForElementPresent | id=results-table setTimeout | 5000
Temporarily increase timeout for a specific operation

Affected Commands

  • Wait commands (waitForElementPresent, waitForVisible, etc.)
  • Page load operations (open, click on links)
  • Implicit waits for elements in commands like click, type

Default Timeouts

  • Page load: 30000 ms (30 seconds)
  • Element location: 10000 ms (10 seconds)
  • Wait commands: 30000 ms (30 seconds)

pause

Basic

Pauses execution for a specified amount of time. Use sparingly, as explicit waits are usually better.

Syntax

pause | milliseconds

Parameters

Target: The pause duration in milliseconds

Examples

click | id=show-animation pause | 2000 assertElementPresent | id=animation-complete
Wait for an animation to complete
forEach | urls | url open | ${url} scrapeCollection | css=.data | data pause | 1000 end
Add a delay between requests to avoid rate limiting

Best Practices

  • Avoid using pause when possible; use explicit wait commands instead
  • Use waitForElementPresent, waitForVisible, etc. to wait for specific conditions
  • Use pause only for cases where explicit waits won't work (animations, rate limiting)
  • Keep pause times as short as possible to avoid unnecessarily slow tests

waitForPageToLoad

Basic

Waits for a page to finish loading. Useful after clicking links or submitting forms.

Syntax

waitForPageToLoad | [timeout]

Parameters

Target: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | linkText=Next Page waitForPageToLoad
Wait for page to load after clicking a link
click | id=submit-form waitForPageToLoad | 60000
Wait up to 60 seconds for a page to load after form submission

Usage Notes

  • This command is automatically used by open and click on links
  • Use this command when automatic waiting is not triggered (e.g., after JavaScript navigation)
  • The command waits for the document.readyState to be "complete"
  • For single-page applications, use waitForElementPresent instead to wait for specific elements

waitForElementPresent

Basic

Waits for an element to be present in the DOM. Does not require the element to be visible.

Syntax

waitForElementPresent | locator | [timeout]

Parameters

Target: A locator that identifies the element to wait for
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=load-data waitForElementPresent | id=data-table
Wait for a table to be loaded after clicking a button
click | id=show-more waitForElementPresent | css=.additional-content | 15000
Wait up to 15 seconds for additional content to appear

Presence vs. Visibility

  • This command only checks if the element exists in the DOM
  • The element does not need to be visible to satisfy this wait
  • Use waitForElementVisible if you need to wait for an element to be visible

waitForElementNotPresent

Basic

Waits for an element to be removed from the DOM.

Syntax

waitForElementNotPresent | locator | [timeout]

Parameters

Target: A locator that identifies the element to wait for
Value: Optional timeout in milliseconds (defaults to current timeout setting)

Examples

click | id=delete-item waitForElementNotPresent | id=item-123
Wait for an item to be removed after clicking delete
click | id=submit-form waitForElementNotPresent | css=.loading-indicator | 20000
Wait up to 20 seconds for a loading indicator to disappear

runScript

Advanced

Executes JavaScript in the context of the current page. Unlike executeScript, this command doesn't return a value.

Syntax

runScript | script

Parameters

Target: The JavaScript code to execute

Examples

runScript | window.scrollTo(0, document.body.scrollHeight);
Scroll to the bottom of the page
runScript | localStorage.clear();
Clear local storage
runScript | document.querySelector('#hidden-button').click();
Click a button that might be difficult to access with regular commands

Usage Notes

  • The script runs in the context of the current page
  • Use executeScript if you need to capture the return value
  • You can access and modify the DOM, variables, and functions on the page
  • Avoid using alert(), confirm(), or prompt() in scripts

executeScript

Advanced

Executes JavaScript in the context of the current page and returns the result.

Syntax

executeScript | script | [variableName]

Parameters

Target: The JavaScript code to execute
Value: Optional variable name to store the result

Examples

executeScript | return document.title; | pageTitle echo | Current page title: ${pageTitle}
Get the page title using JavaScript
executeScript | return document.querySelectorAll('.product').length; | productCount echo | Found ${productCount} products
Count elements using JavaScript
executeScript | return window.innerHeight; | windowHeight executeScript | return document.body.scrollHeight; | pageHeight echo | Window height: ${windowHeight}, Page height: ${pageHeight}
Get window and page dimensions

Return Value Handling

  • The script must include a return statement to return a value
  • Primitive values (strings, numbers, booleans) are returned as-is
  • Objects and arrays are converted to JSON strings
  • DOM elements are converted to their string representation
  • If no variable name is provided, the result is discarded

setWindowSize

Basic

Sets the size of the browser window.

Syntax

setWindowSize | widthxheight

Parameters

Target: The desired window size in the format "widthxheight" (e.g., "1024x768")

Examples

setWindowSize | 1920x1080
Set window size to full HD resolution
setWindowSize | 375x667 open | https://example.com // Test mobile view setWindowSize | 1024x768 // Test tablet view
Test responsive design at different screen sizes

Usage Notes

  • Common resolutions:
    • Desktop: 1920x1080, 1366x768, 1440x900
    • Tablet: 1024x768, 768x1024 (iPad)
    • Mobile: 375x667 (iPhone), 360x640 (Android)
  • The actual viewport size may be smaller due to browser chrome (toolbars, scrollbars)
  • Use executeScript with window.innerWidth and window.innerHeight to get the actual viewport size

maximizeWindow

Basic

Maximizes the browser window to fill the screen.

Syntax

maximizeWindow

Examples

open | https://example.com maximizeWindow
Open a site and maximize the window
