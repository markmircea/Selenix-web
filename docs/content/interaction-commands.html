Interaction Commands

These commands allow you to interact with elements on the page, such as clicking, typing, and selecting options.

click

Basic

Clicks on a target element (e.g., a link, button, checkbox, or radio button).

Syntax

click | locator

Parameters

Target: An element locator that identifies the element to click. Can use id, name, css selector, or xpath (e.g., "id=submitButton", "css=button.primary", "//button[contains(text(),'Submit')]")

Examples

click | id=submit-button click | css=.login-button click | xpath=//button[contains(text(),'Sign In')]
Click on buttons, links, checkboxes, and other interactive elements

Usage Notes

  • If the click is not working as expected, try using clickAtCoordinates instead
  • For elements that need to be visible to be clicked, ensure they are in the viewport
  • Use waitForElementVisible before clicking if elements load dynamically

clickAtCoordinates

Advanced

Clicks at the specified absolute coordinates on the page. Useful for interacting with canvas elements, maps, or complex UI components.

Syntax

clickAtCoordinates | x,y

Parameters

Target: The X and Y coordinates for the click, relative to the viewport's top-left corner (0,0), specified as "x,y" (e.g., "150,200")

Examples

clickAtCoordinates | 320,240 clickAtCoordinates | 100,150
Click at specific coordinates on a map, chart, or canvas element
clickAtCoordinates | 100,150 verifyText | id=status | Selected top section clickAtCoordinates | 100,250 verifyText | id=status | Selected bottom section
Click on different parts of a complex UI component

Coordinate System

  • Coordinates are relative to the viewport's top-left corner (0,0)
  • The x-coordinate increases from left to right
  • The y-coordinate increases from top to bottom
  • For example, 150,200 means 150 pixels from the left edge and 200 pixels from the top edge of the viewport

Best Practices

  • Viewport Considerations: Remember that coordinates are relative to the current viewport. If the page scrolls or resizes, the same coordinates might point to a different element.
  • Responsive Design: For responsive websites, coordinates may need to be adjusted for different screen sizes.
  • Debugging: If your click isn't hitting the right spot, try using the coordinate selector tool again to visually confirm the location.

doubleClick

Basic

Double clicks on an element (e.g., a link, button, checkbox, or radio button).

Syntax

doubleClick | locator

Parameters

Target: An element locator that identifies the element to double-click

Examples

doubleClick | id=edit-field doubleClick | css=.item-row
Double-click to edit text or activate special functionality

Usage Notes

  • If the double-click is not working as expected, try using doubleClickAtCoordinates instead
  • Commonly used for text selection, opening items in file explorers, or activating special features

doubleClickAtCoordinates

Advanced

Double-clicks at the specified absolute coordinates on the page. Useful for interacting with canvas elements, maps, or complex UI components.

Syntax

doubleClickAtCoordinates | x,y

Parameters

Target: The X and Y coordinates for the double-click, relative to the viewport's top-left corner (0,0), specified as "x,y" (e.g., "150,200")

Examples

doubleClickAtCoordinates | 500,300
Double-click on a specific point on a canvas or drawing application
open | https://example.com/map-editor doubleClickAtCoordinates | 320,240 verifyElementPresent | css=.marker
Double-click on a map to place a marker

Coordinate System

  • Coordinates are relative to the viewport's top-left corner (0,0)
  • The x-coordinate increases from left to right
  • The y-coordinate increases from top to bottom

Best Practices

  • Viewport Considerations: Remember that coordinates are relative to the current viewport. If the page scrolls or resizes, the same coordinates might point to a different element.
  • Responsive Design: For responsive websites, coordinates may need to be adjusted for different screen sizes.
  • Maintenance: Coordinate-based commands may require more maintenance than element-based commands if the UI layout changes frequently.

type

Basic

Sets the value of an input field, as though you typed it in. Clears the field first and then enters the specified text.

Syntax

type | locator | value

Parameters

Target: A locator that identifies the element to type into (e.g., id=username, name=password, css=input.search)
Value: The text to type into the element. This can include:
  • Plain text
  • Variables (e.g., ${username})
  • Special keys using the format <KEY> (e.g., <ENTER>, <TAB>, <SPACE>)

Examples

type | id=username | johndoe type | name=password | mySecurePassword123 type | css=input.search | selenium automation
Type text into form fields
type | id=search-box | selenium automation<ENTER> type | id=comment | Line 1<ENTER>Line 2<ENTER>Line 3
Type text with special keys
store | John Doe | fullName type | id=name-field | ${fullName}
Type text using variables
type | css=input[type="file"] | C:/path/to/file.jpg
Upload a file (Chrome only)

Special Features

  • File Uploads: If the target is a file input element (<input type="file">), you can provide a file path as the value to upload a file (Chrome only). XPath locators are not supported for file uploads.
  • Special Keys: Use <KEY> syntax for special keys like <ENTER>, <TAB>, <SPACE>, <CTRL>, etc.
  • Key Combinations: Combine keys with + like <CTRL>+<A> to select all text.
  • If type isn't working properly on JavaScript-heavy pages, try the sendKeys command instead.

sendKeys

Advanced

Simulates keystroke events on the specified element, as though you typed the value key-by-key. Unlike the type command, this does not clear the field first.

Syntax

sendKeys | locator | keySequence

Parameters

Target: A locator that identifies the element to send keys to
Value: The sequence of keys to send, including special keys using <KEY> syntax

Examples

sendKeys | id=editor | This text will be appended sendKeys | css=.rich-editor | <CTRL>+<A><DELETE>New content sendKeys | id=search-input | selenium<ENTER>
Send keystrokes to elements, including special key combinations

When to Use sendKeys vs. type

  • Use sendKeys when you want to append text to existing content
  • Use sendKeys for dynamic UI widgets that require explicit key events
  • Use sendKeys when type doesn't work on JavaScript-heavy pages
  • Use type when you want to replace the existing content completely

check / uncheck

Basic

Check or uncheck a toggle-button (checkbox/radio).

Syntax

check | locator
uncheck | locator

Examples

check | id=agree-terms uncheck | name=newsletter check | css=input[value="option1"]
Handle checkboxes and radio buttons

select

Basic

Select an element from a drop-down menu using an option locator.

Syntax

select | selectLocator | optionLocator

Parameters

Target: Locator for the select element
Value: Option locator (label=, value=, id=, index=) or just the label text

Examples

select | id=country | label=United States select | name=size | value=large select | css=.category-select | index=2 select | id=color | Red
Select options using different identification methods

editContent

Advanced

Sets the value of a content editable element as if you typed in it.

Syntax

editContent | locator | value

Examples

editContent | css=.rich-editor | <h1>Title</h1><p>Content</p> editContent | id=wysiwyg | ${formattedText}
Edit rich text editors and contentEditable elements

dragAndDropToObject

Advanced

Drags an element and drops it on another element.

Syntax

dragAndDropToObject | sourceLocator | targetLocator

Parameters

Target: Locator of the element to be dragged
Value: Locator of the destination element

Examples

dragAndDropToObject | id=draggable-item | id=drop-zone dragAndDropToObject | css=.task-card | css=.completed-column
Drag and drop for kanban boards, file uploads, or sortable lists
