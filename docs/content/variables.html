Variables

Variables in Selenix are powerful containers for storing, manipulating, and sharing data throughout your automations. From simple text storage to complex object manipulation, variables enable sophisticated data processing workflows.

AI-Powered Data Processing

Use the sendToAI command to process variables with natural language instructions like "Extract only prices above $100"

Variable Types

Selenix supports multiple data types for different use cases:

String Variables

Store text, URLs, names, and simple data

store | Hello World | greeting Creates: ${greeting} = "Hello World"

Number Variables

Store numeric values for calculations

store | 42 | answer Creates: ${answer} = 42

Array Variables

Store collections of similar items

scrapeCollection | css=.product-title | productTitles Creates: ${productTitles} = ["Product 1", "Product 2", ...]

Object Variables

Store structured data with multiple fields

storeJson | {"name": "John", "age": 30} | user Creates: ${user} = {name: "John", age: 30}

Boolean Variables

Store true/false values for logic

store | true | isLoggedIn Creates: ${isLoggedIn} = true

Special Variables

Browser states, snapshots, and files

createSnapshot | loginState Creates: Browser state variable

Storing Data in Variables

Multiple commands can store data in variables:

Basic Storage Commands

store Store any text or value
store | My Value | variableName
storeText Extract and store element text
storeText | css=.price | productPrice
storeValue Get input field values
storeValue | name=quantity | currentQty
storeAttribute Extract HTML attributes
storeAttribute | css=img@src | imageUrl

Advanced Storage Commands

scrapeCollection Store arrays of similar elements
scrapeCollection | css=.item | items
scrapeStructured Store complex structured data
scrapeStructured | {"title": "css=.title"} | data
executeScript Store JavaScript execution results
executeScript | return document.title | pageTitle
httpRequest Store API response data
httpRequest | {"url": "api.com/data"} | apiData
