    • executePowerShell

    System Integration

    Execute PowerShell scripts and capture results. Perfect for system operations, file processing, and advanced integrations.

    Syntax

    executePowerShell | scriptPath[|arg1|arg2] | variableName

    Parameters

    Target: Path to PowerShell script, optionally followed by arguments separated by | characters
    Value: Variable name to store the script's output

    Examples

    executePowerShell | C:/scripts/process-data.ps1|${csvFile}|outputDir | processResult executePowerShell | C:/scripts/get-system-info.ps1 | systemInfo executePowerShell | C:/scripts/send-email.ps1|${recipientEmail}|${subject}|${body} | emailResult echo | Script completed: ${processResult}
    Process data files, get system information, and send emails

    PowerShell Integration

    • Scripts receive arguments as $args[0], $args[1], etc.
    • Variable interpolation: ${variableName} is expanded before script execution
    • Return structured data as JSON for easy processing in Selenix
    • Cross-platform: Uses PowerShell Core on macOS/Linux

    echo

    Utility

    Prints the specified message to the log. Essential for debugging and monitoring automation progress.

    Syntax

    echo | message

    Examples

    echo | Starting product scraping process echo | Found ${productCount} products on page ${currentPage} echo | Processing user: ${user.name} (${user.email}) echo | Export completed: ${exportedRecords} records saved
    Log progress, variable values, and completion status

    Debugging Tips

    • Use before/after scraping commands to track progress
    • Include variable values to verify data processing
    • Add timestamps: ${new Date().toISOString()}
    • Use in loops to monitor iteration progress

    debugger

    Development

    Breaks execution and enters debugger mode. Allows you to inspect variables and step through commands manually.

    Syntax

    debugger

    Examples

    scrapeCollection | css=.product | products debugger transformVariable | products | data => data.filter(p => p.price > 100) debugger exportToCSV | products | ./results.csv
    Add breakpoints to inspect data at different stages

    pause

    Utility

    Wait for the specified amount of time before continuing execution.

    Syntax

    pause | milliseconds

    Examples

    click | css=.load-more-button pause | 3000 scrapeCollection | css=.newly-loaded-item | newItems pause | 5000 click | id=submit-form
    Wait for content to load and forms to process

    When to Use Pause

    • After clicking load buttons to wait for new content
    • Between rapid API calls to respect rate limits
    • For slow-loading dynamic content
    • Consider using waitForElementVisible instead when possible

    setSpeed

    Configuration

    Set execution speed by adding delays between commands. Useful for slower websites or debugging.

    Syntax

    setSpeed | milliseconds

    Examples

    setSpeed | 1000 open | https://slow-website.com click | css=.menu-item setSpeed | 0 scrapeCollection | css=.fast-content | data
    Slow down for problematic sites, then speed up for data extraction

    Speed Settings

    • 0: No delay (fastest)
    • 500-1000: Good for most websites
    • 2000+: For very slow or problematic sites
    • Global setting affects all subsequent commands
