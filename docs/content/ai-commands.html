AI & Advanced Commands

These commands leverage artificial intelligence and advanced system integrations to extend your automation capabilities beyond traditional web interactions.

sendToAI

AI-Powered

Leverages AI capabilities to generate or manipulate data during test execution. Process existing variables or generate new content based on your instructions.

Syntax

sendToAI | instructions | variableName

Parameters

Target: Instructions for the AI model, describing what you want it to do. These can include:
  • Data transformation instructions (e.g., "Filter this list to only include items that start with 'A'")
  • Content generation requests (e.g., "Generate a blog post about automation testing")
  • Data analysis directives (e.g., "Analyze this JSON and extract all email addresses")
Value: The name of the variable to process or create:
  • If the variable already exists, the AI will process its content according to your instructions
  • If the variable doesn't exist, the AI will generate new content based solely on your instructions and store it in a new variable with this name

Examples

storeJson | [{"name": "Alice", "age": 25}, {"name": "Bob", "age": 30}, {"name": "Charlie", "age": 35}] | people sendToAI | Filter this list to only include people over 30 years old | people echo | ${people}
Filter existing data using AI
sendToAI | Generate a list of 5 creative product names for a new line of eco-friendly water bottles | productNames echo | ${productNames}
Generate new content with AI
scrapeCollection | css=.product-title | productTitles sendToAI | Convert all these product titles to uppercase and add the prefix "NEW: " to each one | productTitles storeJson | ${productTitles} | formattedTitles
Transform scraped data using AI
sendToAI | Generate a JSON array of 3 fictional user profiles with fields for name, email, age, and occupation | users exportToJSON | users | users.json
Generate structured data with AI and export it

Tips and Best Practices

  • Be specific in your instructions: The more specific your instructions, the better the results
  • Consider data formats: For structured data, specify the format you want
  • Variable types: The command works with any variable type (strings, arrays, objects)
  • Error handling: If the AI encounters an error, it will return an error message that you can check
  • Performance: For large datasets, consider breaking them into smaller chunks
  • Consistency: For repeatable results, use detailed instructions with specific formatting requirements

Common Use Cases

  • Data cleaning and transformation: Normalize data formats, fix inconsistencies
  • Content generation: Create test data, sample content, or placeholder text
  • Data extraction: Pull specific information from complex structures
  • Format conversion: Transform between different data formats
  • Text analysis: Summarize content, extract keywords, or categorize text

executePowerShell

System Integration

Execute PowerShell scripts from within your Selenium IDE tests. This powerful integration enables you to leverage PowerShell's capabilities for system operations, data processing, and automation tasks.

Syntax

executePowerShell | scriptPath[|arg1|arg2|...] | variableName

Parameters

Target: Path to the PowerShell script file, optionally followed by arguments separated by pipe (|) characters.
  • Format: scriptPath[|arg1|arg2|...]
  • Example: C:/scripts/process.ps1|argument1|argument2
Value: Variable name to store the script's output.
  • If provided, the script's output will be stored in this variable for later use in your test.
  • If the script returns JSON or structured data, it will be automatically parsed.

Examples

executePowerShell | C:/scripts/hello.ps1 | result echo | ${result}
Basic usage - run a script and display the result
executePowerShell | C:/scripts/process.ps1|arg1|arg2 | result
Pass arguments to the PowerShell script
store | Hello World | myVariable executePowerShell | C:/scripts/process.ps1|${myVariable}|otherArg | result
Pass variables as arguments to the script
executePowerShell | C:/scripts/getData.ps1 | apiData verify | ${apiData.status} | success
Use the structured result in subsequent commands

PowerShell Script Examples

# Simple script that returns a string (hello.ps1) Write-Output "Hello from PowerShell!"
# Script with arguments (process.ps1) $firstArg = $args[0] $secondArg = $args[1] $result = "Processed: $firstArg, $secondArg" return $result
# Script returning structured data (getData.ps1) $data = @{ "status" = "success" "items" = @( @{ "id" = 1; "name" = "Item 1" }, @{ "id" = 2; "name" = "Item 2" } ) "count" = 2 } return $data

Best Practices

  • Script Path: Use absolute paths to ensure scripts are found regardless of where Selenium IDE is running from.
  • Variable Interpolation: Variables in the format ${variableName} are automatically interpolated before being passed to the PowerShell script.
  • Error Handling: If the PowerShell script encounters an error, the error message will be stored in the result variable.
  • Return Values: PowerShell objects are automatically converted to JSON when returned to Selenium IDE.
  • Security Considerations: Scripts run with the same permissions as the Selenium IDE process.
  • Cross-Platform Compatibility: On Windows, the command uses PowerShell.exe. On macOS/Linux, it uses pwsh (PowerShell Core), which must be installed.

echo

Utility

Prints the specified message to the log. Essential for debugging and monitoring automation progress.

Syntax

echo | message

Examples

echo | Starting product scraping process echo | Found ${productCount} products on page ${currentPage} echo | Processing user: ${user.name} (${user.email}) echo | Export completed: ${exportedRecords} records saved
Log progress, variable values, and completion status

Debugging Tips

  • Use before/after scraping commands to track progress
  • Include variable values to verify data processing
  • Add timestamps: ${new Date().toISOString()}
  • Use in loops to monitor iteration progress

debugger

Development

Breaks execution and enters debugger mode. Allows you to inspect variables and step through commands manually.

Syntax

debugger

Examples

scrapeCollection | css=.product | products debugger transformVariable | products | data => data.filter(p => p.price > 100) debugger exportToCSV | products | ./results.csv
Add breakpoints to inspect data at different stages

pause

Utility

Wait for the specified amount of time before continuing execution.

Syntax

pause | milliseconds

Examples

click | css=.load-more-button pause | 3000 scrapeCollection | css=.newly-loaded-item | newItems pause | 5000 click | id=submit-form
Wait for content to load and forms to process

When to Use Pause

  • After clicking load buttons to wait for new content
  • Between rapid API calls to respect rate limits
  • For slow-loading dynamic content
  • Consider using waitForElementVisible instead when possible

curlRequest

Advanced

Executes a cURL-style HTTP request and stores the response in a variable. Perfect for API testing and data retrieval.

Syntax

curlRequest | curlCommand | variableName

Parameters

Target: A cURL command string or JSON configuration object for the request
Value: Variable name to store the response

Examples

curlRequest | curl https://api.example.com/users | userData echo | Retrieved ${userData.length} users
Simple GET request using cURL syntax
curlRequest | curl -X POST https://api.example.com/login -H "Content-Type: application/json" -d '{"username":"test","password":"password"}' | loginResponse storeJson | ${loginResponse} | loginData store | ${loginData.token} | authToken
POST request with JSON data and header
store | my-api-key | apiKey curlRequest | curl https://api.example.com/data -H "Authorization: Bearer ${apiKey}" | apiData
Using variables in the cURL command

Usage Notes

  • Supports all standard cURL options including headers, request methods, and data
  • Response is automatically parsed as JSON if possible
  • Variables can be interpolated in the cURL command using ${variableName} syntax
  • Useful for API testing, data retrieval, and authentication flows
